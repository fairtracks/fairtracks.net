The FAIRtracks ecosystem

The FAIRtracks ecosystem contains an interconnected group of services built around the FAIRtracks draft standard. The services are aimed at gathering, curating, validating, integrating, indexing, and analyzing the metadata of genomic tracks.

(Please refer to the home page for images of the interconnections between the services, both internally within the FAIRtracks ecosystem, as well as to external services.)

Core services

Search engine for genomic track metadata

Hierarchical browser of the metadata attributes and all values actually present in the database
Categorical search
Graphical builder of SQL-based search queries
Browse results in table or hierarchical form
Export results in JSON or GSuite formats
REST API for integration into downstream tools and scripts

Metadata augmentation service

RESTful API for automatized augmentation of minimum information
Import a FAIRtracks-annotated JSON file with a minimal set of fields
Export a JSON file with augmented human-readable fields of relevance of downstream users
Automated retrieve of most recent ontology versions
Automated generation of human-readable ontology labels

Metadata validator service

RESTful API for validation of JSON files against the FAIRtracks schemas
Implementation based on standard JSON Schema validation libraries with additional custom extensions
Validation of ontology terms against specific ontology versions
Validation of CURIEs against entries of the Identifiers.org Resolution Service
Support for document-set restrictions e.g. "unique" constraints enforcement and "foreignProperty" checks

FAIRtracks JSON to GSuite format conversion service

GSuite is a tabular text format describing a collection of genomic tracks
GSuite contains information about track pre-processing, possible data analysis strategies, and reference genome
RESTful API for conversion of FAIRtracks JSON files into the GSuite format
Enabling the TrackFind client in the GSuite HyperBrowser to output track metadata in the GSuite format
Making track collections from particular search queries consumable by manipulation and analysis tools in the FAIRtracks ecosystem

Connected services

Statistical analysis framework for track data

The first general framework for statistical analysis of genomic tracks
Created at the crossroads of biology, statistics, and computer science
Open-ended web server, based on the Galaxy web framework
Can analyze genomic variation, gene regulation, chromatin structure, 3D genome organization, and more...
Hundreds of precise tools and analyses
Prototype track search and import tool that inspired FAIRtracks is being replaced by TrackFind client

A global centralized collection of publicly accessible track hubs

A common point of entry to register data collections into Ensembl and UCSC genome browsers
The exchange format is a set of text files collectively referred to as a track hub
Each track is assigned information such as its URL and reference genome. Additional attributes, including experimental metadata, can be attached
We propose highlighting FAIRtracks-compliant entries and making them easily available to downstream tools
RESTful API for remote content querying

The BLUEPRINT/EPICO Data Analysis Portal

Enables exploration of the behaviour of candidate genes, pathways or genome regions across the BLUEPRINT datasets
Provides insightful views of genomic tracks based on various analyses
Supports different reference genomic assemblies and organisms
Designed following the client-server paradigm, which supports different pluggable data backends at its REST API
Integration with the FAIRtracks ecosystem via a dedicated backend enabling interoperability of queries and data model with the Track Hub API
